Shortcuts

Typing in 'lambda' on any input field will automatically replace it with 'λ'.

Syntax

The syntax in the code is very sensitive. Global definitions are always automatically applied, but if you want to write your own application from scratch inside the expression input field, you'll need to explicitly define the function being applied.

- typing in an expression such as 'add 1 2' will automatically convert it to '(λm.λn.λf.λx.((m f) ((n f) x))) 1 2'.

- however, the interpreter will not see any application if you type in 'λm.λn.λf.λx.((m f) ((n f) x)) 1 2', meaning it will be ignored.

So, remember, always wrap your explicit definitions around parentheses if you want to avoid issues!

Beware the "Y"!

Looking at the pure untyped lambda calculus paradigm, you might ask how does one get around loops? We naturally think first in terms of either self-referential functions or for/while-loops (recursive vs iterative approaches), however, in pure untyped lambda calculus we support neither self-referencing, nor is there any native support for iterations (goto functions). Instead, we have to get creative.
We use a trick called a fixed-point combinator. This is how we define The Y fixed-point combinator: Y = λf.(λx.(f (x x))) (λx.(f (x x)))
The combinator works by passing a function as its own argument, which creates a cycle that simulates recursion. Our interpreter natively stores a Y combinator in its global definitions that can be called at any point in the code, featuring as well an implementation of Y for a factorial function.
But, be very mindful when using this function. It is very dangerous, and can very easily lead to infinite recursion! Whenever you must use the Y combinator, make sure to add a reachable base case.
